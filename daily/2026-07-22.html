GitLab 预认证反序列化 RCE：PoC 公开 24 小时即在野，企业 DevOps 平台再成突破口

CVSS 9.8 在野利用 · 已发布补丁

GitLab CE/EE 的专案导入功能在处理导出包时存在不安全反序列化（CWE-502）：攻击者向 /api/v4/projects/import 上传特制 .tar.gz 导出包，恶意 gadget 链在反序列化阶段触发命令注入（CWE-94），于无需认证下获得命令执行。官方 7/21 发布补丁（17.1.6/17.2.4/17.3.1），7/22 多个情报源确认在野探测与利用。

漏洞机制/攻击方式：

  • 类型：不可信数据反序列化 + 命令注入 (CWE-502 / CWE-94)
  • 攻击方式：构造含恶意 gadget 链的专案导出包，经 import 端点上传触发
  • 影响范围：所有未打补丁的 GitLab CE/EE 实例（含自托管分支）
  • 后果：预认证 RCE、读源码仓库、窃取 CI/CD 令牌、横向至内网构建节点

值得关注的原因：

  • "披露即武器化"窗口压缩至 24 小时，WAF 规则来不及分发
  • DevOps 平台托管大量源码与部署凭据，单点失守即全局沦陷
  • 大量实例暴露在公网，自动扫描即可批量收割

vLLM 供应链污染：恶意 pip 包 vllm-serve 伪装官方，安装即 RCE

PyPI 恶意包 · AI 推理基础设施 7/22 披露 · 建议锁定版本

PyPI 出现名为 vllm-serve 的恶意包，命名刻意模仿主流推理框架 vLLM，在 setup.py 安装钩子植入反向 Shell 与凭证窃取逻辑；安装该包的 GPU 节点立即回连攻击者 C2 并导出环境变量中的 API Key / 模型权重路径，攻击面从"模型文件"下沉到"推理依赖链"。
