1 严重

FortiSandbox 三重临界漏洞在野利用：安全基础设施反向成入口

CVE-2026-39808 / 25089 / 39813 在野 · KEV 期限 7/19

本周多家安全厂商（Defused Cyber、National CERT Pakistan、Hive Security）报告，攻击者已对 Fortinet FortiSandbox / Cloud / PaaS 的多个临界漏洞发起真实利用。CISA 在 7/16 更新中将 CVE-2026-39808 与 CVE-2026-25089（CVSS 9.1，未认证 OS 命令注入）纳入 KEV，联邦机构须在 7/19 前修复；同簇 CVE-2026-39813（JRPC API 路径遍历认证绕过，部分源评 9.8）本周首次被观测到武器化。三者均无需认证、无需用户交互。

漏洞机制/攻击方式：

  • 类型：CVE-2026-39808/25089 为 OS 命令注入（CWE-78），CVE-2026-39813 为路径遍历认证绕过（CWE-22/306）；均可达设备完全接管。
  • 攻击方式：CVE-2026-39808 经 jid GET 参数注入以 root 执行命令（4 月起已有公开 PoC + Nuclei 模板）；CVE-2026-39813 经 JRPC API ../filedir 越权；蜜罐捕获来自 141.11.43.175（AS136510）对 /jsonrpc/ 的畸形 JSONRPC POST。
  • 影响范围：暴露在互联网管理接口的 FortiSandbox / FortiSandbox Cloud / FortiSandbox PaaS。
  • 后果：沙箱放行恶意样本、伪造"已检测"结论，并以其 Security Fabric 集成（防火墙/邮件网关/SIEM）作内网横向跳板。

值得关注的原因：

  • FortiSandbox 是"负责保护企业的安全设备"，沦陷后防御信任受损，反成最高效入侵入口。
  • 三漏洞均零前置条件，暴露管理接口即"扫描即被打"；Fortinet 设备长期是攻击者重点目标（4 月 CVE-2026-35616 FortiClient EMS 同源在野先例）。
  • 补丁早已发布（4 月）却仅 KEV 后才被重视，提示"已发补丁 ≠ 已缓解"，须以资产暴露面驱动排查。
攻击者 探测 Web UI/JRPC 命令注入/认证绕过 设备接管/root 沙箱放行+内网横向

SharePoint CVE-2026-58644（CVSS 9.8 反序列化 RCE）零日入 KEV

CVSS 9.8 · CWE-502 反序列化 零日 · 期限 7/19

CISA 于 7/16 将 CVE-2026-58644 加入 KEV，联邦机构须在 7/19 前修复。该漏洞为反序列化 of untrusted data，CVSS 9.8（AV:N/AC:L/PR:N/UI:N），微软在 7/14 Patch Tuesday 发补丁后将公告修订为已确认在野——即零日武器化先于补丁。影响 SharePoint SE/2019/2016，而 2016/2019 恰于 7/14 终止扩展支持。

3 高危

F5 披露 NGINX 三枚高危漏洞（CVE-2026-42533 堆溢出 9.2）

CVSS 9.2 / 8.8 / 8.3 边界/入口层

F5 于 7/15 披露影响 NGINX Plus 与 NGINX Open Source 的三枚高危漏洞，覆盖 Ingress Controller、Gateway Fabric、App Protect WAF、Instance Manager。最严重 CVE-2026-42533 为 map 指令正则处理触发的 worker 进程堆缓冲区溢出（CWE-122），CVSS v4.0 9.2；在禁用/绕过 ASLR 的系统上可能 RCE。第二漏洞为 ngx_http_slice_module 未初始化内存泄漏（8.8），第三为 ngx_http_ssi_module 释放后使用（8.3）。

攻击方式：特制 HTTP 请求命中 map 正则/未命名捕获/SSI+proxy_pass 组合 → 内存破坏、崩溃或潜在代码执行；缓解为使用命名捕获、关闭非必要模块，但根本修复仍需升级。

4 高危

Linux 内核 authencesn 权限提升 PoC 公开（CVE-2026-31431）

本地提权至 root PoC 已公开

研究员公开 CVE-2026-31431 利用 PoC：Linux 内核 authencesn（AF_ALG 密码接口）长度校验缺陷，使 dirty page 被 write-back 到攻击者无权限写入的文件（如 /etc/passwd），实现本地提权至 root。PoC 于 6.12.85 内核验证，思路与 Dirty COW 家族的"页缓存写回"技术相近。

值得关注：PoC 可复现，在共享主机/容器宿主/CI Runner 直接危害；常作为容器逃逸、客户机逃逸跳板，与 VMSwitch 9.9 叠加风险放大；终端检测须覆盖异常 page write-back 与 /etc/passwd 改写。

5 中高危

Mistune 路径遍历 CVE-2026-59924，波及 AI/LLM 工具链

CVSS 5.9 · CWE-22 供应链/AI

Python Markdown 解析库 Mistune（< 3.3.0）的 Include.parse() 拼接用户提供的 include 路径时未校验结果是否仍在预期目录内，导致路径遍历（CWE-22）：经 md.read() 处理不可信 Markdown 时可读取目录外文件。GHSA-r4rv-85jg-w4mf，NVD CVE-2026-59924，已在 3.3.0 修复。

值得关注：大量 AI/LLM 工具链、RAG 管线、文档问答、CMS 后端依赖 Markdown 渲染，不可信内容经 Mistune 处理即触发，是"AI 应用供应链"中被低估的攻击面；须做 SBOM 级排查升级 ≥3.3.0。
