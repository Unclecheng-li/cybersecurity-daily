Cisco Secure Workload CVE-2026-20223 — CVSS 10.0 满分漏洞

CVSS 10.0 满分漏洞 无需认证

攻击者无需任何认证即可通过构造 REST API 请求获取 Secure Workload 平台的站点管理员（Site Admin）权限。Secure Workload 是 Cisco 零信任微隔离方案的核心组件，管理员权限意味着攻击者可完全控制多租户微隔离策略。

攻击流程：

攻击者 未认证 REST API 请求 Secure Workload API 漏洞: 无鉴权 获取 Site Admin 权限 篡改微隔离策略
字段详情
CVECVE-2026-20223
CVSS10.0（满分）
影响产品Cisco Secure Workload（原 Tetration）
影响版本3.9 及以前、3.10、4.0（含 SaaS 和本地部署）
修复状态已修复，SaaS 用户自动更新

值得关注的原因：

  • 满分 CVSS + 零认证 + 核心安全产品，三重叠加罕见
  • 微隔离策略篡改可直接导致横向移动畅通无阻
  • SaaS 用户虽不受影响，但本地部署版（金融机构/政府常见）需手动升级
2 严重

Apache OFBiz CVE-2026-45434 — 三个缺陷组合实现认证绕过→RCE

CVSS 8.8 单请求RCE 受影响：Apache OFBiz < 24.09.06

这是 Apache OFBiz 认证流程中三个缺陷的链式组合，一次 HTTP POST 即可实现 RCE。

三个缺陷：

缺陷#1: checkLogin()仅检查"error"返回值 缺陷#2: requirePasswordChange从HTTP请求读取 缺陷#3: ProgramExport无权限检查无沙箱 GroovyShell.evaluate()执行任意代码

攻击时序：

POST /control/ProgramExport requirePasswordChange=Y LoginWorker checkLogin() 仅检查"error"，放行 无权限检查，无沙箱 RCE (uid=0 root)

值得关注的原因：

  • 单请求 RCE，利用门槛极低
  • OFBiz 默认附带 10+ 演示账户（密码均为 ofbiz），大量部署未改默认凭证
  • 与 2023 年 CVE-2023-51467（CVSS 9.8）利用同一 requirePasswordChange 逻辑缺陷——同类漏洞再次出现
  • 建议：升级至 24.09.06+，禁用默认演示账户

OpenClaw AI "Claw Chain" 四漏洞链 — 24.5 万 AI Agent 服务器暴露

最高 CVSS 9.6 AI基础设施 公网暴露 24.5 万台

四个漏洞可链式利用形成完整攻击链——"Claw Chain"：

CVE缺陷类型攻击效果
CVE-2026-44112沙箱执行时序缺陷绕过安全校验
CVE-2026-44115命令校验绕过执行未授权命令
CVE-2026-44118权限越权提升至服务器权限
CVE-2026-44113路径读取绕过窃取密钥与敏感文件

攻击链：

CVE-2026-44112 沙箱时序缺陷 CVE-2026-44115 命令校验绕过 CVE-2026-44118 权限越权 CVE-2026-44113 路径读取绕过 服务器完全控制 持久化后门植入

值得关注的原因：

  • AI Agent 框架安全首例大规模披露——AI 基础设施安全盲区
  • 四漏洞组合 = 从低权限到完全控制的完整攻击链
  • 24.5 万公网暴露实例，实际受影响面可能更大
  • AI Agent 持久化后门威胁：篡改 AI 行为比传统后门更难检测
4 高危

Claude Code 沙箱绕过 — AI 编程助手的隐蔽风险

暴露时长：5 个月+ AI安全 修复版本：v2.1.90+

安全研究员 Guan 发现 Claude Code 的网络沙箱存在严重缺陷——沙箱内的任何进程都可以通过 SOCKS5 代理静默绕过出口限制，访问被封锁的主机。

攻击流程：

恶意提示词/依赖投毒 Claude Code 沙箱内进程 SOCKS5 绕过 突破出口限制 外传 API 密钥/源代码/环境变量

值得关注的原因：

  • AI 编程助手已成为开发者日常工具，沙箱安全是最后的防线
  • Anthropic 标记为"重复"而非主动披露，透明度严重不足
  • 与 OpenClaw 事件叠加，AI 工具链安全已成系统性问题
  • 建议：升级到 v2.1.90+ 并轮换所有凭证

CISA 承包商 GitHub 泄露 844MB 凭证 — 安全机构翻车

泄露机构：CISA 暴露6个月 发现方：GitGuardian

CISA 一家承包商将名为 "Private-CISA" 的公开 GitHub 仓库暴露近 6 个月，包含 844MB 敏感数据——其中 importantAWStokens 文件包含 3 个 AWS GovCloud 管理员凭证。

时间线：

2025-11-13 承包商创建仓库 持续暴露约 6 个月 2026-05-14 GitGuardian 发现泄露 2026-05-15 仓库下架 部分 AWS 密钥仍然有效
字段详情
泄露内容844MB：AWS GovCloud 管理员密钥、明文密码、GitHub Token、K8s 配置
暴露时长6 个月（2025.11.13 → 2026.5.14）
密钥轮换仓库下架 48 小时后部分 AWS 密钥仍然有效

值得关注的原因：

  • 讽刺到极致：负责全美网络安全的 CISA，自己犯了最低级的凭证泄露错误
  • 暴露时间 6 个月 + 下架后密钥 48 小时仍有效 = 攻击窗口巨大
  • 供应链角度：触及 CISA 构建环境 → 类 SolarWinds 场景
  • 第三方风险管理：CISA 对外包的安全管控形同虚设

NYC Health+Hospitals — 180 万患者含生物识别数据泄露

影响人数：180 万 生物识别泄露 驻留时间：77 天

纽约市公立医疗体系通过第三方供应商被入侵，攻击者在系统内驻留长达 77 天。最严重的是泄露数据包含不可重置的生物识别信息（指纹、掌纹）——密码可以改，指纹不能换。

受害机构NYC Health + Hospitals（纽约市公立医疗体系）
影响人数180 万
入侵方式第三方供应商被入侵
泄露数据医保信息、诊疗记录、社保号、银行卡 + 指纹/掌纹
发现时间2026.02.02

值得关注的原因：

  • 生物识别数据泄露不可逆，影响终身
  • 第三方供应商风险再次成为主要攻击面（与 Verizon DBIR 数据一致）
  • 77 天驻留时间远超行业平均检测时间
  • 该机构 2026 年第二起第三方关联泄露
