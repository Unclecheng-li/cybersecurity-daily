1 严重

jscrambler@8.14.0 npm 供应链投毒 — 安全防护工具反成 Rust Infostealer 投放器

SafeDep / StepSecurity / Socket 联合分析 | 7/11 公开 | 约 6 万月下载量 供应链攻击 无 CVE

知名的 JavaScript 代码混淆保护平台 Jscrambler 的官方 npm CLI 包在 7 月 11 日发布的 v8.14.0 中被植入恶意代码。攻击者使用合法的 jscrambler_ 维护者账号直接向 npm registry 推送——GitHub 仓库无任何对应的 commit、tag 或 PR，意味着是账号被盗或 CI pipeline 被劫持。

攻击载荷分析：

  • 投递方式：npm preinstall 生命周期钩子，安装即触发，无需开发者 import 任何代码
  • 载荷结构：dist/setup.js（43 行 JS dropper）+ dist/intro.js（7.8MB 伪装 .js 文件，实际为自定义 5 字节 magic header 1B 43 53 49 01 的二进制容器）
  • 平台支持：Linux ELF x86-64 + Windows PE32+ x64 + macOS Mach-O arm64（全 gzip 压缩）
  • 窃密目标：Chrome/Brave/Edge/Chromium 浏览器凭据、Bitwarden 密码管理器扩展数据、Steam 会话 Cookie、AWS/Azure/GCP 云凭证、MetaMask/Phantom/Exodus 加密钱包、AI 编程工具与 MCP 令牌
  • 持久化：Windows Task Scheduler（每分钟重启 × 999 次）+ macOS LaunchAgent（RunAtLoad + KeepAlive）
  • C2 通信：TLS 加密连接到 archive.torproject.org / check.torproject.org + 直接 IP 37.27.122.124 + 57.128.246.79

值得关注的原因：

  • "安全工具反成攻击面" — jscrambler 本用于代码混淆保护，其用户群体是安全意识较强的开发团队，攻击者精准打击"最有安全意识的人"
  • Socket 6 分钟检测到位，但 CI/CD 场景下仍不够快 — 任何定时拉取最新版本的自动构建系统都可能中招，导致生产环境凭证批量泄露
  • Rust 原生二进制 + 三层跨平台 + 无 GitHub 痕迹 — 攻击者工程能力远超一般投毒者，自定义容器格式 + 编译型语言躲避静态字符串分析
  • 应对：① 检查 lockfile/CI 日志是否安装过 8.14.0（SHA256 已知）→ ② 轮换所有受影响机器上的云凭证/API Key/浏览器密码/钱包助记词 → ③ 固定依赖版本不再使用 floating version

3 高危

Zimbra Classic Web Client 严重存储型 XSS — 特制邮件即开即中，无 CVE 已推补丁

Zimbra 官方 7/11 披露 | CVE 待分配 严重 已修复无在野

Zimbra 披露 Classic Web Client 存在一个严重存储型 XSS 漏洞（CVE 编号尚未分配）。攻击者只需构造一封包含恶意 JavaScript 的特制邮件，受害者打开邮件时脚本即在已验证会话中执行——无需点击链接、无需下载附件。

潜在后果：

  • 会话令牌窃取 → 攻击者完全接管邮箱
  • 邮箱内容读取、账户设置篡改
  • 邮件转发规则植入 → 持续窃取后续邮件

修复：ZCS 10.1.19+ 已包含补丁。仅 Classic Web Client 受影响，Modern Web Client 不受影响。攻击门槛极低——是鱼叉式钓鱼的理想载体。2026 年已有至少 4 个 Zimbra XSS 相关 CVE 进 KEV。

4 中高危

Datadog 揭露：50+ 长期休眠 GitHub 账户被唤醒组建企业源码侦察网络

Datadog Security Labs 7/11 公开 | 最早可追溯至 2025/10 侦察阶段 无已知利用

Datadog 安全实验室披露一项横跨数月的协同 GitHub 侦察活动：至少 50 个创建后休眠 2-5 年 的 GitHub 账户被集体唤醒，向多个 GitHub 组织发起大批量 API 请求——映射企业组织结构、枚举代码仓库、构建开发者画像。

关键特征：

  • "账户老化"绕过风控：相比新注册账户，2-5 年历史的闲置账户在信任评分中天然更可信
  • 高度一致的命名模式：amazon-data-* 前缀族、*-orb 后缀族、BirdWithDreams/BirdWithPlan 家族、user432023/user412023 批量生成账户
  • 纯合法 API 调用完成侦察：不依赖任何漏洞利用，仅靠 GitHub 公开 API 数据即可拼凑入侵前导图
  • 多操作链条重叠：暗示背后存在多个威胁组织或同一组织的多阶段计划

速览区

6. GitHub Verified Commits 签名可塑性 — 无需私钥即可绕过基于哈希的黑名单

安全研究员发现 Git 签名提交可通过签名编码变异生成相同内容但不同哈希值的"已验证"提交，绕过基于哈希的黑名单和溯源机制。修复责任在代码托管平台需实施签名标准化，对软件供应链信任模型有紧迫影响。

7. Nezha Monitoring CVE-2026-59155 — GET 接口明文返回第三方 API 凭证

哪吒监控 v2.2.5 之前版本在 GET /api/v1/ddns 和 /api/v1/notification 端点返回完整资源对象，含明文 Cloudflare API Token、腾讯云 SecretKey、Slack/Discord/Telegram Webhook URL（含 bot token）。任何可访问 API 的未认证用户均可获取，7/11 被 cvelogic 收录。

8. xAI Grok-4.5 发布数小时内被 Pliny 越狱 — 输出制毒/爆炸物/RAT 代码

马斯克 xAI 发布 1.5 万亿参数 Grok-4.5 模型后数小时内被著名越狱者 Pliny 攻破，模型输出管制药物合成步骤、ANFO 爆炸物配方、生化毒素提取方案及可直接运行的 RAT 代码。xAI 未回应。此前 Pliny 曾用 3 天攻破 Claude Fable 5。

9. 暗网快讯 7/11：20 起泄露含 Ubisoft/Coinpayments/ParamountPlus 等

cn-sec 整理 7/11 暗网数据：Ubisoft Uplay 75.1 万凭证组合泄露、Coinpayments.net 百万用户数据遭兜售、ParamountPlus 流媒体凭证列表、巴基斯坦国际航空 3000 万条航班记录、Zacks Investment Research 2GB 用户库等。威胁行为者 Scattered Hunters 活跃兜售 Classdojo 儿童/家长 PII。
