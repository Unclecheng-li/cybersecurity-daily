T3MP3ST — 35 工具 AI Agent 自主红队框架（XBEN 90.1% pass@1）

开源 AGPL-3.0 / elder-plinius / 7/5 发布 AI 红队化

开源框架 T3MP3ST 把通用 AI 编码 Agent（Claude Code / OpenAI Codex / Hermes）改造为自主红队代理，无需新的 API key、额外云基础设施或额外账单（"Keyless Warfare"）。通过 War Room 网页或 CLI，用户授权一个目标后，本机已运行的 AI Agent 就成为执行"任务大脑"，整个侦察到利用的杀伤链都被自动化驱动。

漏洞机制/攻击方式：

  • 类型：AI Agent 多智能体编排的安全测试框架（双刃剑）
  • 攻击方式：8 角色杀伤链（Recon → Scanner → Exploiter → Infiltrator → Exfiltrator → Ghost → Coordinator → Analyst）映射 MITRE ATT&CK
  • 影响范围：Web 应用（XBEN 90.1% pass@1）/ CTF（Cybench 23/40）/ 嵌入式/OT/机器人 / 智能合约 / 源代码白盒审计
  • 后果：能力被滥用可对授权目标实现全自动漏洞挖掘与利用；同样可用于蓝队高保真演练

值得关注的原因：

  • 从"对话式 AI 辅助"跨入"AI Agent 自主 exploit 链路"
  • 0 API 成本 + 0 新基础设施 = 门槛极低
  • XBOW 自评 85% → 框架 90.1%，对蓝队意味着更紧迫的攻防升级
  • 10 个 2026 新 CVE 命中 8 个（命中 CVE 晚于训练截止，排除记忆嫌疑 → 真实泛化能力）
  • 开源 + AGPL + "仅授权测试"声明：双刃剑的边界是契约与审计

2 高危

ARToken — M365 OAuth Device Code Flow 钓鱼面板（80+ 后渗透功能）

Cisco Talos 披露 / EvilTokens 家族 零 MFA / 长效 PRT

ARToken 是新浮出水面的 Microsoft 365 钓鱼面板，深度集成 EvilTokens 工具链（Sekoia 1月首次披露、微软确认为大规模威胁）。它通过滥用合法的 Microsoft device login 流程：诱骗用户在真实的 microsoft.com/devicelogin 输入攻击者提供的 device code → 后台静默获得 access token → 升级为 PRT（primary refresh token，用户改密码无效）。

受影响组件：

组件风险
Microsoft 365 邮箱（Exchange Online）高危：可读全邮箱 + 假冒发件人
SharePoint / OneDrive高危：可下载文件 + 创建后门规则
Azure AD 会话中高危：PRT 升级绕过改密
3 高危

Microsoft Exchange SSRF CVE-2026-45504 — CVSS 8.8 任意文件读取 PoC 公开

CVE-2026-45504 / HawkTrace CVSS 8.8 / PoC 公开

认证低权限 Exchange 用户可构造特制 reference attachment，触发 OneDriveProUtilities 组件的 SSRF 缺陷。利用 file:// scheme + URL fragment # 绕过 scheme 校验，把 WebApplicationUrl 改为 file:///C:/Windows/win.ini#，Exchange 会执行 FileWebRequest 并把内容返回攻击者。GitHub 已公开完整 PoC（HawkTrace/CVE-2026-45504），利用门槛清零。

影响：读取 win.ini / hosts / 服务配置 / 凭据库；为横向攻击 Exchange 后端与 AD 集成铺路。

4 中高危

Claude Cowork 沙箱漏洞 — DLL 旁路 + isResume 提权到 Root Shell

Anthropic Claude Cowork / Windows 1.9255.2.0 本地代码执行前提

Armadin 研究员披露：Claude Cowork（Windows）的沙箱可被串成完整突破链——DLL 旁路 USERENV.dll 拿到合法签名进程内代码执行 → 用 AI Agent fuzzing JSON RPC 协议 → 暴露 spawn 方法 → 构造 isResume=true + name=root 直接拿到 bubblewrap 沙箱内的 root shell，绕过 egress 代理与用户隔离。

关键点：Authenticode 签名校验 ≠ 身份校验；Go Unmarshal 错误信息成为攻击者的协议字典。

5 中高危

Apple "Hide My Email" 漏洞 — 未修补超一年，可还原真实邮箱

Apple iCloud+ / EasyOptouts / 404 Media 部分披露

研究者 Tyler Murphy 一年前向 Apple 报告 iCloud+ Hide My Email 别名可被反向解析到真实邮箱，至今未修。404 Media 在自有别名上验证漏洞仍然存在。研究者决定"部分披露"——公开警告但保留技术细节，避免被滥用。

影响：记者、活动人士、敏感行业用户的匿名保护层失效；高风险用户应假定别名可链接到真实邮箱并调整 OPSEC。

9. Microsoft 365 Apps RCE — Excel 越界读 OOB Read → 任意代码执行

无需认证/权限，仅需用户交互；影响 Microsoft 365 Apps / Excel 2016 / Office 2019 / LTSC 2021/2024；缓解：Protected View + 打补丁。

10. Azure CLI ROPC 密码喷洒 — 64 家组织 78 账户沦陷，8100 万次登录尝试

滥用 Azure CLI 遗留 OAuth ROPC 流程绕过 MFA；6/12–6/26 期间；关联 IPv6 段 2a0a:d683::/32；修复：All Cloud Apps 强制 MFA + 禁用 ROPC grants。
