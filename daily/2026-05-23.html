Trend Micro Apex One CVE-2026-34926：端点管理面漏洞已被在野利用

在野利用 端点管理 JPCERT/CC 更新告警

JPCERT/CC 更新 Trend Micro 多产品漏洞告警，其中 CVE-2026-34926 为 TrendAI Apex One (On Premise) 的相对路径遍历漏洞。Trend Micro 已确认该漏洞存在在野利用。

攻击流程：

攻击者获得认证入口 利用CVE-2026-34926相对路径遍历 篡改Apex One管理服务器文件 构造恶意Agent更新或任务 下发至受管终端 批量执行恶意代码

值得关注的原因：

  • Apex One 属于企业端点安全管理系统，一旦管理服务器被控，影响不是单点主机，而是整批受管终端
  • 该漏洞已出现真实攻击，不是理论风险
  • 端点安全产品具备高权限和广泛部署特征，被攻破后容易变成"合法分发恶意代码"的通道
  • 建议：按 Trend Micro 官方公告打补丁，限制管理端访问

3 严重

Drupal CVE-2026-9082：PostgreSQL 场景下的高危 SQL 注入

SA-CORE-2026-004 SQL注入 PostgreSQL + JSON:API/Views

Akamai 分析称，CVE-2026-9082 是 Drupal Core 的关键 SQL 注入漏洞。风险主要集中在使用 PostgreSQL 后端数据库，并依赖 JSON:API、Views 暴露过滤器、Entity autocomplete 等端点的 Drupal 站点。

攻击过程：

攻击者构造query string数组参数 PHP参数解析保留可控array key Drupal EntityQuery将key传入查询条件构造 PostgreSQL Driver placeholder名称处理异常 形成SQL注入 数据泄露

值得关注的原因：

  • Drupal 是常见 CMS，政企、教育、内容站点中仍有大量部署
  • PostgreSQL + JSON:API/Views 这类组合在真实站点中并不少见
  • Akamai 指出补丁发布后攻击代码通常会快速出现
  • 建议：立即升级 Drupal Core，排查异常数组参数请求

HPLIP CVE-2026-8631 / CVE-2026-8632：Linux 打印栈可导致提权或任意代码执行

CVSS 9.3 / 8.5 RCE/提权 修复版本：HPLIP 3.26.4

HP Linux Imaging and Printing Software（HPLIP）两个漏洞，可能通过命令注入或缓冲区溢出触发。

CVECVSS严重性利用条件影响
CVE-2026-86319.3Critical网络可利用、低复杂度、无需权限、无需交互权限提升 / 任意代码执行
CVE-2026-86328.5High本地可利用、低权限、无需交互权限提升 / 任意代码执行

值得关注的原因：

  • 打印服务常被当作"低优先级资产"，但在办公网、服务器和桌面 Linux 中都有部署
  • CVE-2026-8631 为网络可利用且无需权限，适合被内网横向利用
  • 打印链路通常日志与检测不足，容易成为隐蔽入口
  • 建议：升级 HPLIP 至 3.26.4，限制打印服务暴露
5 高危

Linux Kernel CVE-2026-46333：本地低权限 shell 到 root / 凭据泄露路径

Qualys TRU 披露 本地提权 PoC 已出现

Qualys TRU 披露 CVE-2026-46333，这是 Linux Kernel ptrace 路径中的逻辑缺陷。攻击者在获得本地低权限 shell 后，可结合 pidfd_getfd() 捕获特权进程的文件描述符或认证 IPC 通道。

可造成的影响：

  • 读取 /etc/shadow
  • 泄露 SSH host private keys
  • 通过 pkexec、accounts-daemon 等目标以 root 执行命令
  • 在多租户主机、CI runner、开发机、容器宿主机上扩大入侵影响

值得关注的原因：

  • "本地漏洞"不等于低优先级
  • SSH host key 泄露会影响主机身份信任和后续横向移动
  • 公开 PoC 已出现，补丁窗口应压缩
  • 建议：应用内核安全更新，临时缓解：设置 ptrace_scope=2
6 高危

Fox Tempest：Microsoft 摧毁"恶意软件签名即服务"基础设施

超过 1000 个代码签名证书 关联勒索生态

Microsoft Threat Intelligence 披露并干扰 Fox Tempest 运营的 Malware-signing-as-a-Service（MSaaS）。该组织滥用 Microsoft Artifact Signing 生成短生命周期代码签名证书，为恶意软件提供"可信签名"。

维度数据
创建证书超过 1000 个代码签名证书
Azure租户数百个
已吊销证书超过 1000 个
被签名恶意软件Oyster/Broomstick、Lumma Stealer、Vidar 等
关联勒索生态Rhysida、INC、Qilin、Akira、BlackByte 等

值得关注的原因：

  • 代码签名本应代表软件发布可信链，但被服务化滥用后，会降低"签名=安全"的判断价值
  • 攻击者可将恶意安装包伪装为 Teams、AnyDesk、PuTTY、Webex 等常见软件
  • 建议：不要把"文件有有效签名"作为放行依据，启用 SmartScreen、云端防护
7 中高危

First VPN 被跨国执法摧毁：匿名化基础设施成为勒索生态打击重点

Operation Saffron 25 个勒索软件组织使用 33 台服务器被扣押

法国、荷兰牵头，Europol、Eurojust、FBI 等参与的 Operation Saffron 摧毁了面向网络犯罪的 First VPN Service。该服务至少被 25 个勒索软件组织用于网络侦察、入侵、勒索攻击等活动。

维度数据
行动时间2026 年 5 月 19 日至 20 日
扣押服务器33 台
出口节点32 个，分布在 27 个国家
关联用户506 名

值得关注的原因：

  • 这是对勒索攻击链"匿名化支撑层"的打击，不只是关闭一个网站
  • 犯罪 VPN、代理、bulletproof hosting 正成为勒索生态中的关键基础设施
  • 用户身份被执法掌握，会削弱这类"无日志、不配合司法"的地下服务可信度
  • 建议：将公开出口节点 IP 纳入威胁情报回溯

VulnCheck：2026 年"常规被攻击漏洞"加速，边界设备与开发者生态继续升温

VulnCheck 报告 25 个 Routinely Targeted 漏洞 KEV 增长 59%

VulnCheck 截至 2026 年 5 月 21 日识别出 25 个 Routinely Targeted 漏洞。其 KEV 列表今年已新增 394 个有野外利用证据的 CVE，较 2025 年同期增长 59%。

  • 企业边界设备：Ivanti EPMM、BeyondTrust、Cisco SD-WAN/FMC、cPanel & WHM

  • AI / 自动化 / 开发者平台：Langflow、n8n、MCPJam Inspector、marimo
