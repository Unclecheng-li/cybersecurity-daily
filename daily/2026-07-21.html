ServiceNow AI 平台预认证沙箱逃逸 RCE：补丁 4 天即在野，攻击者绕开公开 PoC

CVSS 9.5 / 9.8 在野利用 · 已发布补丁

ServiceNow AI Platform 的 /assessment_thanks.do 预认证端点存在沙箱逃逸 RCE：GlideRecord 查询将用户可控输入传入查询函数，特制 JavaScript 在查询执行前被求值；攻击者借 gs.include() 加载受限脚本库、操纵其全局对象，使代码在更宽松上下文编译执行，逃出沙箱获得原语，进而读任意表、建管理员、接管 MID Server。平台支撑 85% 财富 500 强。

漏洞机制/攻击方式：

  • 类型：GlideRecord 查询代码注入 + 脚本沙箱逃逸 (CWE-94)
  • 攻击方式：向 /assessment_thanks.do 发特制请求，注入恶意 JS 并经 gs.include() 绕过沙箱限制
  • 影响范围：Australia Patch 2 及更早、Yokohama/Zurich/Brazil 多发行版未打补丁实例
  • 后果：预认证 RCE、读全表、建管理员、经 MID Server 横向至企业内网

值得关注的原因：

  • "披露即武器化"窗口压缩到 4 天，且攻击者走不同沙箱逃逸 gadget，只封堵 PoC 无效
  • 云实例 24h 修复、本地实例需手动推进强制阶段，暴露窗口被拉长
  • MID Server 是云与企业内网之桥，SaaS 突破影响远超实例本身
攻击者 触发点 漏洞利用 RCE执行 目标沦陷
