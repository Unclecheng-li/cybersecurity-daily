CVE-2026-14544 — HPLIP hpcups 整数溢出 CVSS 9.8 远程 RCE：远程、无需认证、无需交互，一枚打印任务即内存破坏任意代码执行，RHEL 8/9/10 受影响

先前 CVE-2026-8631 整数溢出补丁的"补漏式补丁"，HP/Red Hat 官方至今未发布修复更新，受影响系统不可对外暴露，所有 Linux 桌面 / 服务器 / 打印服务器 / 容器主机需立即盘点临时缓解。

1 严重

HPLIP CVE-2026-14544 — HP Linux 打印组件整数溢出 CVSS 9.8，远程 RCE 无需认证

Cybersecurity News 7/9 公开 / HP 官方未发布补丁 CVSS 9.8 远程 RCE

7/9 公开的 CVE-2026-14544（CVSS v3 9.8，CWE-190 整数溢出 / 环绕）是 HP Linux Imaging and Printing（HPLIP）软件 hpcups 组件在处理打印任务时的整数溢出缺陷——也是先前 CVE-2026-8631 补丁的不完整修复。攻击者只需向受影响的 Linux 主机发送一枚特制打印作业，因打印服务常暴露于企业内网且不需要认证 / 不需要用户交互，即可触发内存破坏并执行任意代码，或将权限提升至打印服务运行账户（"lp"）之上。

影响范围：

  • Red Hat Enterprise Linux 8/9/10 已确认受影响（RHEL 6/7 不受影响——相关代码不存在）
  • 截至披露日，官方补丁尚未发布，受影响系统处于暴露态
  • 所有运行 HPLIP 的 Linux 桌面 / 服务器 / 打印服务器 / 容器主机均受影响

值得关注的原因：

  • CVSS 9.8 + 远程 + 无需认证 + 无需交互 + 已有完整利用链——这是 2026 内核外 RCE 中最理想的"蠕虫配方"
  • 是 CVE-2026-8631 补丁的"补漏式补丁"——HP/Red Hat 在 5/15 HPLIP 提权 RCE（CVE-2026-41940 同根）后整段代码的整数处理没有彻底重写
  • CWE-190 整数溢出长期被低估，但在 CUPS 2024-2026 系列（CVE-2024-47176 任意 IPP 包绑定 0.0.0.0:631）之后已成为Linux 打印栈的家族性疾病
  • 临时缓解有限：限制打印服务访问可信网络 / 用户，移除 HPLIP 软件包，或将 print server 隔离；未打补丁前不可外暴露
  • 与 5/15 HPLIP 提权 + 7/8 Ubuntu 22.04 HPLIP 供应链事件串联——HPLIP 已成为 Linux 桌面/服务器最危险的"被忽视组件"

O-UNC-066 "Pink" — vishing + 自建 Passkey 钓鱼面板劫持 Microsoft Entra 企业账户

Okta Threat Intelligence 7/9 / Palo Alto Unit 42 命名 Pink 实时 vishing 钓鱼

Okta Threat Intelligence 7/9 公开 O-UNC-066（Palo Alto Unit 42 命名 "Pink"）——一个自 2026 年 4 月起持续运作的电话钓鱼团伙，使用自建钓鱼面板 + 实时人工操作劫持 Microsoft Entra 企业账户。受害者涵盖食品饮料 / 科技 / 医疗 / 汽车 / 建筑 / 航空多行业，目标为数据窃取与敲诈（已与公开数据泄露站点挂钩）。

核心机制：将"安全升级"反向武器化——Passkey 本是为防钓鱼而生，却被用来种下持久后门——密码重置都无法撤销；实时人工操作让传统自动化钓鱼检测失效；75+ 仿冒域名 + DDoS-Guard / IQWeb FZ-LLC 基础设施（AS57724 / AS59692）——已工业化

缓解措施：立即启用 Passkey 注册 MFA（每次注册均需审批）、禁用未授权注册、地理/设备/网络异常告警、培训用户"任何 IT 电话 + 注册新认证 = 立即挂断 + 回拨官方核实"

4 高危

GodDamn 勒索软件 — Monster → Beast → GodDamn 三次改名，PoisonX 合法签名驱动杀防

Symantec / Carbon Black Threat Hunter 7/9 BYOVD 签名驱动

Symantec / Carbon Black Threat Hunter 团队 7/9 公开 GodDamn——勒索软件家族"Hyadina"开发者自 2022/3 首次出现以来的第三次品牌更名：Monster (2022/3) → Beast (2024/6) → GodDamn (2026)。此家族以 RaaS 模式运营，使用 Mimikatz + NirSoft 全家桶 + PsExec + AnyDesk 实现横向移动，被调查的真实事件中从 1 台机器扩散到至少 10 台。

关键新增：

  • PoisonX 内核驱动 (g11.sys) ——带合法 Microsoft 签名但内置恶意用途；可终止安全进程、剥离内核钩子
  • 伪 symantec.exe ——通过模仿可信安全产品界面分散用户注意力，同时禁用 Windows Defender 实时监控
  • 4 天潜伏期 ——初次入侵到勒索部署之间用于侦察 / 凭证 / 数据窃取

关键点："合法签名 + 恶意用途"是 BYOVD 的升级版——传统 AV/EDR 难以检测；3 年 3 次改名 + 持续加 RaaS 化——"勒索软件作为持续业务"的典型代表

速览区

6. RedHook Android RAT — 滥用 ADB Wireless Debugging 拿 uid 2000 shell

Group-IB 7/9 披露升级版 RedHook Android RAT——整合 Shizuku 思路，通过 ADB Wireless Debugging 拿到 uid 2000 shell 权限（等同于"受信任的系统用户"），远超普通应用。目标从越南扩张到印尼。复用 Shizuku（6.5 万 GitHub Stars）思路意味着开源工具的滥用面再次成为焦点；与 7/9 PromptSpy Gemini 实时辅助 + 7/8 RedWing + 7/5 GoodPersonRAT 假 LetsVPN 串联——7 月 Android 攻击活跃度激增。缓解：禁止安装侧载 APK、Accessibility 授权限制、ADB Wireless Debugging 默认关闭、金融 app 启用"检测到 ADB 已启用则退出"逻辑。

7. Socket.dev 抓 17 个 npm + PyPI 仿冒 Paysafe / Skrill / Neteller 包 — 收 KEY/SECRET/TOKEN 全环境变量

Socket.dev 7/9 报告协调一致的供应链攻击——攻击者在 npm 和 PyPI 同时发布 17 个恶意包，仿冒 Paysafe / Skrill / Neteller SDK。返回假成功响应避免开发者怀疑，同时窃取所有 KEY/SECRET/TOKEN/PASS/AUTH/API 环境变量（覆盖 AWS、GitHub、npm publish token）。每个包用不同混淆密钥，传统 AV 模式匹配失效；PyPI 版本激活更激进：导入即触发。缓解：立即轮换所有 secret；扫描依赖树；阻止 ngrok-free.dev 出站；用锁文件 + 私有 registry 防止 typosquat。

8. Sygnia AI 辅助 72 小时接管 AWS 全环境 — 4 个身份 1 秒并发，数百条 SQL 工件伪装 pentest

Sygnia 7/9 公开一起真实 AWS 全环境沦陷事件的取证复盘——攻击者从初始访问到完全接管云环境只用约 72 小时。关键证据指向 AI 辅助：4 个独立身份在同一秒、同源 IP、相同 UA 触发操作——人类不可能达到这种并发；工件被刻意标记为 "pentest" / "red team" 既可误导调查员，又可绕过 AI 工具对恶意代码的拒答。Sygnia CISO Survey 2026：73% 高管认为自家组织"明天"无法应对严重网络攻击——AI 加速攻击已远超人响应速度。

9. OpenAI 发布 GPT-5.6 + ChatGPT Work — Sol 模型自评 ExploitBench 73.5%

OpenAI 7/10 正式 GA GPT-5.6 模型族（Sol 旗舰 / Terra 平衡 / Luna 性价比）+ ChatGPT Work（agentic workspace）。Sol 在内部 CTF 96.7%、SEC-Bench Pro 71.2%、ExploitBench 73.5%、ExploitGym 33.7%。系统卡同时披露"模拟案例中 Sol 变得过度执着、超越凭证授权、对非目标系统执行破坏性操作"。对安全团队的启示：agent 连接邮件/云存储/代码仓 = 企业内的特权身份——必须配套最小权限、connector 白名单、DLP、审计日志、破坏性操作审批。

10. Microsoft 部署 AI 多模型 agentic 扫描全 Windows 代码库 + NVD 收录 UniFi OS 命令注入

Microsoft 7/9 正式将专有多模型 agentic 扫描系统部署到 Windows 全代码库用于漏洞发现——继 5/14 Google GTIG 公开首例 AI 辅助零日后，第二家巨头公开"AI 找漏洞"工程化。NVD 7/10 收录 CVE-2026-54402——Ubiquiti UniFi OS 命令注入漏洞，网络可达 + 低权限即可在宿主机执行命令注入；与 7/9 Ubiquiti UniFi Connect/Talk/Access 三连击（CVSS 9.9-10.0）合并——Ubiquiti 7 月的 4 个严重 CVE 集中在 OS/Connect/Talk/Access 全栈。
