WordPress wp2shell 预认证 RCE 漏洞链：两枚"缝隙"组合成匿名接管

CVE-2026-63030 + CVE-2026-60137 PoC 公开 · 现在野探测

7/17 WordPress 紧急发布 6.8.6/6.9.5/7.0.2/7.1-beta2 并启动强制自动更新，修复由 Searchlight Cyber（Assetnote）Adam Kues 披露、代号 wp2shell 的两枚漏洞。二者单看都"可控"，链式组合即成未认证远程代码执行：无需登录、无需插件、无需用户交互，默认安装即可被打。截至 7/19 完整机理与可用 PoC 已在 GitHub 流通，PatchStack 报告 7/17 晚已观测到在野活动。

漏洞机制/攻击方式：

  • 类型：CVE-2026-60137 为 WP_Query 的 author__not_in 参数 SQL 注入（9.1，回溯至 6.8）；CVE-2026-63030 为 REST /batch/v1 路由混淆（7.5，6.9 起引入）。
  • 攻击方式：批量端点用两个并行数组跟踪子请求，某子请求 wp_parse_url() 解析失败时数组错位一格致"串岗"，嵌套后绕过方法白名单，把攻击者输入喂入 author__not_in（传字符串跳过数组校验）触发匿名盲注。
  • 影响范围：6.8.0-6.8.5 仅 SQLi（修 6.8.6）；6.9.0-6.9.4 与 7.0.0-7.0.1 为完整 RCE 链。WP 承载全球超 5 亿站点，公网仍测得数千万实例。
  • 后果：拖库读取管理员哈希 → 离线爆破 → 登录上传恶意插件/WebShell → 站点完全接管。

值得关注的原因：

  • "评分陷阱"教科书案例：单 CVE 按 CVSS triage 会把 7.5 排低、把 9.1 当"需插件先决条件"，两条都判错——风险恰在缝隙里。
  • 补丁即武器：修复 commit 公开后全球同步逆向，PoC 已出，大规模利用高度可期。
  • 强制更新≠已修复：自动更新可能失败或回滚，须服务端版本+checksum 实测；WAF 需同时拦 /wp-json/batch/v1 与 ?rest_route=/batch/v1。
匿名攻击者 /batch/v1 路由混淆 匿名 SQL 注入 拖库爆破管理员 RCE 站点接管

Kyverno CVE-2026-54523（CVSS 9.6）集群权限提升 PoC 公开

CVSS 9.6 · 跨命名空间提权 PoC 公开

Kubernetes 原生策略引擎 Kyverno 曝严重漏洞，命名空间租户可提权为任意命名空间（含 kube-system）管理员。根因：NamespacedGeneratingPolicy 的 CEL 生成器 apply() 接受目标命名空间参数，但未校验其是否与策略自身命名空间一致；后台控制器默认持集群级权限，恶意租户构造策略令其在目标命名空间创建 RoleBinding 即完成提权。

受影响组件：

组件风险
Kyverno ≤ 1.18.1严重 · 集群管理员提权
NamespacedGeneratingPolicy 创建者高危 · 需审查撤权
Kyverno 1.18.2已修复 · 建议升级
3 高危

ViteVenom：7 个 npm 包用区块链四层 C2 投递 RAT，溯源朝鲜

npm 供应链 · SuccessKey/PolinRider 导入时执行

Checkmarx 7/17 披露 ViteVenom：7 个仿冒 @vitejs 作用域的 npm 包（6/29-7/3 发布，合计 2,400+ 下载）在导入时（非安装时）执行，投递具反弹 shell、凭据窃取、文件外泄、持久化后门的 RAT。C2 为四层区块链（Tron/Aptos/BSC）+ HTTP 回落（198.105.127[.]210），指针存公链不可被查封删除。

溯源：Checkmarx 归因 SuccessKey（与 ChainVeil 共享 Tier-2 钱包/XOR 密钥）；opensourcemalware 进一步比对 TRON 钱包、Aptos 地址、XOR 密钥与朝鲜 Lazarus 的 PolinRider 逐字节一致，累计归因 4,295 恶意资产。防御要点为依赖清单盘点+导入路径审计+凭据轮换。

4 严重

Inc 勒索利用 SonicWall SMA1000 零日 — CVE-2026-15409/15410

SSRF + 代码注入链 在野勒索 · KEV

Inc 勒索团伙正积极利用 SonicWall SMA1000 两枚零日：CVE-2026-15409（未认证 SSRF 探测内网）+ CVE-2026-15410（需认证代码注入取 root），链式控制后窃凭据、建持久化、向域控横移。两枚均在 CISA KEV，联邦大限 7/17 已过，SonicWall 已发热修补丁。

值得关注：打补丁≠清除威胁——已入侵设备须完整取证、重装镜像、重置凭据；边界 VPN/远程接入设备"披露即武器化+勒索接管"态势延续，未修补 SMA1000 应默认视为已沦陷。

5 严重

FortiSandbox / SharePoint KEV 大限今日到期，在野确认

39808/25089 9.1 · 58644 9.8 KEV 大限 7/19

CISA 7/16 将 FortiSandbox CVE-2026-39808/25089（均 9.1 未认证命令注入）与 SharePoint CVE-2026-58644（9.8 反序列化 RCE 零日）纳入 KEV，整改大限均为今日 7/19。同簇 FortiSandbox CVE-2026-39813 本周亦遭利用。

值得关注：FortiSandbox 补丁 4/6 月即发却直到 KEV 才被重视，"已发补丁≠已缓解"；SharePoint 攻击链伴随窃取 IIS machine key 可"补丁后仍持久化"，须补丁+轮换密钥+清痕迹+启用 AMSI。安全基础设施自身失守反成入侵入口。
