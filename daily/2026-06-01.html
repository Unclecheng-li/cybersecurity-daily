CVE-2026-41089 — Windows Netlogon 零点击RCE确认在野利用

CVSS 9.8 在野利用 补丁：已发布（2026年5月Patch Tuesday）

Windows Netlogon服务中存在栈缓冲区溢出漏洞（CWE-121），继2020年Zerologon（CVE-2020-1472）之后，Netlogon协议再次曝出"核弹级"漏洞。攻击者无需任何凭据、无需用户交互（零点击），仅需向域控制器发送一次特制RPC请求，即可获得SYSTEM权限，完全接管整个Active Directory域环境。

关键技术要点：

项目详情
漏洞类型栈缓冲区溢出（Stack-Based Buffer Overflow）
触发点NetrServerAuthenticate3 RPC函数的ComputerName参数未做长度校验
攻击门槛极低：单次恶意RPC请求，预认证、零点击
影响版本Windows Server 2012 / 2012 R2 / 2016 / 2019 / 2022（全系列）
补丁编号KB5089549 / KB5089550 / KB5089551 等

与Zerologon对比：

Zerologon (2020) 加密缺陷 重置域控密码 概率性成功
CVE-2026-41089 (2026) 栈溢出 直接RCE 单次尝试100%成功

值得关注的原因：

  • CyberSecurityNews于6月1日报道该漏洞已确认在野利用
  • 相比Zerologon需要多次尝试，新漏洞单次请求即可成功
  • 企业域控制器是最高价值目标，一旦被利用意味着整个AD域沦陷
  • 所有暴露在互联网的域控制器应在48小时内完成修补

CVE-2026-0257 — PAN-OS GlobalProtect VPN认证绕过

CVSS 7.8 CISA截止日：今日 状态：在野利用中

Palo Alto Networks PAN-OS的GlobalProtect门户和网关存在认证绕过漏洞。攻击者通过伪造认证重写cookie，可绕过VPN认证建立未经授权的连接，直接访问企业内网。

攻击时间线：

2026-05-29CISA加入KEV目录
2026-06-01联邦机构修复截止日

值得关注的原因：

  • 今日（6月1日）是CISA对联邦机构的强制修复截止日
  • Rapid7已确认两波在野利用，攻击者成功获取了企业VPN内网访问权限
  • 大量企业使用PAN-OS作为远程接入方案
  • 临时缓解：禁用Authentication Override功能或为该功能生成独立证书

CVE-2026-8398 — Daemon Tools Lite供应链攻击：300万台终端中招

严重程度：Critical CISA KEV 发现方：卡巴斯基GReAT团队

老牌虚拟光驱工具DAEMON Tools Lite遭遇供应链攻击。攻击者渗透开发商AVB Disc Soft的构建基础设施，篡改官方安装包并使用被盗的合法EV代码签名证书对恶意文件重签名。

攻击链路：

渗透AVB Disc Soft内网 获取构建服务器权限 盗取EV代码签名证书 篡改核心二进制文件 使用合法证书重签名 通过官网分发 300万台终端感染
项目详情
攻击窗口2026年4月8日 ~ 5月5日（28天）
受污染版本12.5.0.2421 ~ 12.5.0.2434（5个版本）
恶意组件DTHelper.exe / DiscSoftBusServiceLite.exe / DTShellHlp.exe
C2域名env-check.daemontools.cc（3月27日注册）
影响范围100+国家，约300万台终端
精准打击仅对约12台高价值目标投放QUIC RAT高级后门
修复版本12.6.0.2445（5月8日发布）

值得关注的原因：

  • 合法EV证书签名的恶意软件可绕过所有传统安全检测
  • 20年以上历史、5亿次累计下载的老牌工具被精准投毒
  • 供应链攻击已成为顶级威胁向量
  • 建议：排查是否安装了12.5.0.2421~12.5.0.2434版本

Gogs自托管Git零日RCE — 无补丁、PoC已公开、5万+企业暴露

CVSS 9.4 零日无补丁 CVE编号：暂未分配

Rapid7于5月28日披露了Gogs自托管Git服务中的严重零日漏洞。攻击者仅需拥有普通用户账号，通过创建包含恶意参数的分支名，即可在服务器上执行任意代码。

漏洞机制：

注册普通账号 创建恶意分支名 发起Pull Request 选择变基并合并 参数注入触发 任意命令执行
项目详情
漏洞类型参数注入（CWE-88），非传统命令注入
缺陷位置internal/database/pull.go → Merge()函数
根本原因git rebase命令拼接用户可控的pr.BaseBranch参数
影响版本全版本（含最新稳定版0.14.2和开发版0.15.0+dev）
暴露规模公网1100+实例（Shodan），企业私有部署5万+
报告→披露时间3月17日报告 → 5月28日公开（72天无补丁）

值得关注的原因：

  • 典型的"修复延迟"案例——等待72天后仍无补丁
  • PoC已公开且Metasploit模块可用
  • 在野利用已确认（挖矿木马XMRig植入、暗网售卖访问权限）
  • 建议：禁用"允许合并前变基"功能，或限制注册权限

CVE-2026-48027 — Nx Console VS Code扩展供应链投毒

CVSS 9.8 CISA KEV 攻击窗口：仅18分钟

Nx Console（Nx/Lerna的VS Code UI扩展）在VS Code Marketplace和Open VSX注册表中被注入恶意版本18.95.0。该版本在UTC 12:30~12:48的18分钟窗口内通过VS Code自动更新机制分发。

攻击过程：

攻击者发布恶意版本18.95.0 VS Code自动推送更新 后台自动更新Nx Console 触发扩展执行恶意payload 外传凭证+环境变量
项目详情
恶意版本18.95.0（12:30 PM ~ 12:48 PM UTC，仅18分钟）
分发渠道VS Code Marketplace + Open VSX
攻击载荷获取混淆payload → 窃取凭证/环境变量/源码
影响范围启用了Nx Console自动更新的所有VS Code用户
发现时间2026年5月19日

值得关注的原因：

  • VS Code Marketplace的信任机制再次被利用
  • 18分钟的攻击窗口虽然短暂，但自动更新机制意味着大量开发者可能在不知情的情况下安装了恶意版本
  • 开发者工具链已成为2026年最受关注的攻击面
  • 建议：检查Nx Console版本是否曾为18.95.0
