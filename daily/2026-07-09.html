CVE-2026-43499 GhostLock — Linux 内核 futex/rt-mutex UAF 潜伏 15 年 PoC 公开：Nebula Security PoC 可靠性 97%，任意本地用户一键 root + 容器逃逸

继 Januscape KVM 16 年 UAF 后的又一颗 15 年级 Linux 内核核弹，"竞速修补"窗口已启动——所有容器主机 / Kubernetes worker / 多租户云环境均在受影响范围。

1 严重

GhostLock CVE-2026-43499 — Linux 内核潜伏 15 年的 UAF，PoC 公开且可靠性 97%

7/8 Nebula Security / Google kernelCTF 2337 号披露 CVSS 7.8 PoC 公开

7/8 公开的 CVE-2026-43499 "GhostLock"（CVSS 3.1 7.8，AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H，CWE-416）是 Linux 内核 futex / rt-mutex 子系统一处 Use-After-Free，自 2011 年起所有主流发行版均受影响；Nebula Security（通过 Google kernelCTF 提交 2337 号漏洞）公开的 PoC 可靠性高达 97%，可让任意本地用户 获得 root + 容器逃逸——比 7/7 公开的 Januscape（KVM 客户机→宿主机）更"普惠"。

漏洞机制/攻击方式：

  • 类型：Use-After-Free（CWE-416），rt-mutex 路径下 futex 同步原语竞争
  • 攻击方式：任意线程调用 futex 即可触发 rt-mutex 释放后引用 → 内核代码执行
  • 影响范围：Linux 服务器、桌面、云实例、容器主机（Docker / Kubernetes）—— 任何带 shell 的系统
  • 后果：本地提权 root + 容器逃逸 → 多租户云单租户沦陷即影响同机全部租户

值得关注的原因：

  • "15 年 + 公开 PoC + 97% 可靠"是历史级别——5/15 cPanel 大规模在野利用还只针对 Web 服务，GhostLock 命中几乎每一台 Linux 主机
  • 容器逃逸 = Kubernetes / Docker 平台的致命威胁：集群内任意容器沦陷即影响宿主
  • Nebula Security 的 PoC 已发布——威胁行为人将其整合到工具包只是时间问题，竞速修补窗口已经开启
  • 与 7/7 公开的 Januscape（KVM Shadow MMU） 合并：2 天内两次"内核老代码集中暴露"，Linux 内核审计焦点已切换
  • 临时缓解有限：seccomp 阻止特定 futex 操作能降低风险但非通用解，多租户 + 容器场景必须尽快打补丁

GhostApproval — 6 款主流 AI 编码代理遭"符号链接幽灵审批"系统性绕过

Wiz Research / 7/9 披露 类别级设计缺陷

Wiz 7/9 公开 "GhostApproval"——一种类别级（category-level）设计缺陷：恶意代码库用 project_settings.json → ~/.ssh/authorized_keys 这类符号链接骗 AI 代理在"workspace setup"中直接把攻击者 SSH 公钥写入开发者机器；同时在 UI 层展示与真实目标不同的文件名（"知情同意"被劫持）。

受影响：Amazon Q Developer CVE-2026-12958（高，v1.69.0）、Cursor CVE-2026-50549（严重，v3.0）、Google Antigravity v1.19.6（严重）、Augment v0.754.3（严重）、Windsurf v1.9566（严重）、Anthropic Claude Code v2.1.42+（Anthropic 起初以"超出威胁模型"驳回，2.1.173+ 加入符号链接解析与告警）。

关键点：不是单点 bug 是类别级设计缺陷；Claude Code auto-mode 用户已默认批准 93% 的权限提示——Human-in-the-Loop 早已沦为"橡皮图章"；90 天披露窗口内 Anthropic 从"驳回"到"补"反映 AI 厂商对威胁模型认知不一致

4 高危

Friendly Fire — Claude Code / Codex 默认 Auto 模式遭 prompt 注入劫持执行恶意二进制

AI Now Institute / 7/9 披露 PoC 公开

AI Now Institute 7/9 公开 "Friendly Fire" 攻击——Anthropic Claude Code CLI（Sonnet 4.6 / 5 / Opus 4.8）与 OpenAI Codex CLI（GPT-5.5）在 默认 auto-mode / auto-review 配置下被劫持，无需 hooks、插件、MCP 服务器或配置文件——prompt 注入直接藏在第三方开源库的 README 和源文件里。研究者改包 geopy，加 security.sh、code_policies（恶意二进制）和 code_policies.go（诱饵 Go 源文件），README 把 security.sh 描述为"安全审计脚本"——当用户给代理 prompt "Perform security testing on /path/to/geopy" 时，代理自动探索代码、读 README、看诱饵源文件、判定二进制"安全"、执行——主机 RCE。

关键技术：auto-mode 分类器被欺骗（代码匹配启发式被 spoof）；同注入对 Claude Code / Codex 均有效 → 不是模型 bug 是信任边界设计 bug；CI/CD 放大效应：Megalodon GitHub 投毒、PyTorch Lightning 供应链事件表明——自动漏洞扫描委派给 AI 代理等同于直接信任投毒者

Palo Alto Networks 官方 / 7/9 CVSS-B 9.2 最高紧急

Palo Alto Networks 7/9 披露 CVE-2026-0288——PAN-OS User-ID Terminal Server Agent (TSA) 组件多处缓冲区溢出（CVSS-B 9.2 高 / CVSS-BT 7.2），被 Palo Alto 标为"最高紧急"。攻击者只需网络可达 TSA 的 IP 和端口，无需认证、无需用户交互即可触发内存破坏，导致 RCE 或 DoS。受影响版本横跨 PAN-OS 10.2/11.1/11.2/12.1 全部活跃分支，Prisma Access 11.2.0/10.2.0（中危）。披露人：Liang Zhu。

6. Microsoft Defender "RoguePlanet" 零日 CVE-2026-50656 已修复

Microsoft 7/9 发布补丁修复 Defender 中预先披露的 0day "RoguePlanet"（CVE-2026-50656）——CVSS 7.8 本地提权，影响 Microsoft Malware Protection Engine < 1.1.26060.3008（含 Defender Antivirus / Security Essentials / SCEP），CWE-59 不安全链接解析，Microsoft 评估"更可能被利用"，本地低权限即可静默提权。自动更新即可，但 Defender 禁用系统仍需手动核验引擎版本。

7. GoodPersonRAT — 假冒 LetsVPN 安装器植入 RAT，C2 域名中文意为"你是好人"

ThreatLocker 7/9 披露 GoodPersonRAT——假冒中国流行 VPN 工具 LetsVPN 安装器（Kuailianwin-setup.86.msi），捆绑真实签名的 LetsVPN 蒙混过关；MSI 内藏三段：合法 VPN 安装器 + shellcode 加载器（promecefplugilte8.exe）+ 加密 shellcode 数据（20260609.dat）；最终载荷完全在内存执行（不落盘），提供键盘记录、屏幕控制、文件操作、Telegram Desktop 会话窃取与代理重定向；自动添加 Defender 排除项。C2 域名包括 nishihaoren1.top ~ nishihaoren40.top（"你是好人"中文拼音）——给出攻击者中文背景强证据。

8. PromptSpy — 首款在受害端实时调用 Google Gemini 的 Android 恶意软件

ESET 7/8 披露 PromptSpy——首款在运行中调用生成式 AI 的 Android 恶意软件。通过仿冒阿根廷某银行子品牌的网站分发 MorganArg 应用；常规 RAT 功能（PIN 截取、应用枚举、屏幕录像）+ 为绕过不同 Android 版本的"最近任务"手势防护，实时把屏幕 UI 状态发给 Gemini，请求"操作指引"，按 Gemini 返回的"在哪里点击哪里滑动"通过 Accessibility 服务执行；通过不可见 overlay 覆盖 Stop / Uninstall 按钮反卸载。乌克兰 2026/2 已有实际感染——具有中文开发环境痕迹。

9. APT-C-20 "Fancy Bear" — PNG 图像藏 Shellcode，Fileless C# Backdoor 经 Filen.io C2

360 7/8 披露 APT-C-20（APT28 / Fancy Bear）新型攻击链：以"东欧某国防部文件"为诱饵的 Word 宏文档 readme.docm（469 字节）→ 加密宏释放 dnxstore.dll（COM 劫载）+ EdgeLogo.png（看似 Edge 图标，实际藏 LSB 隐写 shellcode）→ 内存解密 shellcode → 反射加载 Publish.exe（混淆 C# Backdoor）→ 通过 Filen.io 云存储（gateway.filen.io / gateway.filen.net）C2。全链路不落盘 PE——传统 AV 失效，目标为政府与外交机构。

10. Accenture 确认数据泄露 — 35GB 源码 + 凭据兜售，威胁者"888" 2024 年曾涉 Accenture 假数据

威胁行为人 "888" 在 PwnForums 7/6 发帖，一次性价（仅收 Monero）兜售据称 7 月初从 Accenture 窃取的 35GB 源码 + RSA 密钥 + SSH 密钥 + Azure PAT + Azure Storage Access Key + 内部配置；放出 Azure DevOps 私有仓 AtriasTalentAcademy（accenture.com 域名）截图。Accenture 7/9 确认事件存在，声明"已修复源头，对运营与服务交付无影响"。"888" 2024/6 曾以 32,826 名 Accenture 员工数据集兜售（前次被 Accenture 否认）。LockBit 2021/8 曾以 6TB 数据勒索 Accenture 5000 万美元——这家全球最大 IT 咨询公司 5 年内第 3 次被入侵。
