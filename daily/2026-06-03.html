CVE-2026-41089 — Windows Netlogon 零点击 RCE 持续在野利用：继 Zerologon 后第二个核弹级漏洞

1 严重

CVE-2026-41089 — Windows Netlogon 零点击 RCE 持续在野利用

CVSS 9.8 在野利用 披露时间：2026年5月12日（微软5月补丁日）

CVE-2026-41089 是继 Zerologon（CVE-2020-1472）之后，Windows Netlogon 服务曝出的第二个核弹级漏洞。攻击者可通过 UDP 389 端口发送恶意 CLDAP 查询，触发栈缓冲区溢出，实现远程代码执行。

漏洞机制：

  • 类型：栈缓冲区溢出 → 零点击 RCE
  • 攻击向量：UDP 389端口（CLDAP查询）
  • 认证要求：无需认证
  • 点击要求：零点击
  • 后果：以SYSTEM权限运行，域控制器接管

与 Zerologon 对比：

值得关注的原因：

  • 继 Zerologon 后第二个核弹级 Netlogon 漏洞：同为零点击、无需认证、CVSS 9.8
  • 在野利用已确认：比利时网络安全中心（CCB）于6月1日发布紧急警报
  • 域控制器一击即溃：成功利用可直接接管 Windows 域控制器，整个 Active Directory 域沦陷
  • 补丁已发但部署滞后：5月12日补丁已发布，但企业域控制器补丁部署通常较慢

防御建议：立即应用微软5月补丁日更新，监控UDP 389端口异常流量，限制域控制器网络暴露面。

攻击者发送恶意CLDAP查询 UDP 389端口触发栈溢出 Netlogon服务内存损坏 远程代码执行以SYSTEM权限运行 域控制器接管 整个AD域沦陷
2 严重

CVE-2024-21182 — Oracle WebLogic 两年前修补漏洞现被在野利用

严重程度：Critical 披露时间：2024年7月首次披露 CISA KEV收录

CVE-2024-21182 是 Oracle WebLogic Server 中的一个未授权访问漏洞，允许未经身份验证的攻击者通过 T3 或 IIOP 协议访问并接管 WebLogic 服务器。

漏洞要点：

  • 类型：未授权访问（T3/IIOP协议）
  • 状态：在野利用已确认
  • CISA收录：2026年6月1日加入KEV目录
  • 影响：Oracle WebLogic 广泛用于金融、政府、电信等行业

值得关注的原因：

  • 两年后被利用：漏洞披露两年后仍被攻击者利用，说明"已修补"不等于"已部署"
  • CISA KEV 强制修复：美国联邦机构必须在截止日前完成修复
  • 攻击门槛低：无需认证，仅需网络可达即可利用

防御建议：检查 WebLogic 版本，应用 2024 年 7 月补丁，限制 T3/IIOP 协议对外暴露。

3 高危

Red Hat npm 包供应链攻击 — 32 个官方包被入侵

严重程度：High 披露时间：2026年6月1日 影响：@redhat-cloud-services 命名空间下 32 个 npm 包

2026年6月1日，Red Hat 官方 npm 命名空间下的 32 个包被供应链攻击入侵，攻击者注入了 96 个恶意版本。此次攻击分发了一种名为 "Miasma" 的凭证窃取恶意软件变种。

攻击手法：

  • 凭证窃取蠕虫（类似 Mini Shai-Hulud）
  • 自动窃取 npm、GitHub 和云服务凭证
  • 可自动感染其他仓库，具有自我复制能力

值得关注的原因：

  • 官方命名空间被入侵：Red Hat 是全球最大的开源软件供应商之一
  • 自动传播蠕虫：恶意软件具有自我复制能力
  • 凭证窃取范围广：涵盖 npm、GitHub、AWS、Azure 等主流平台

防御建议：检查项目依赖中是否包含受影响版本，使用 npm audit 扫描，启用包完整性校验。

4 严重

CVE-2025-48595 — Google Android 零日漏洞在野利用

严重程度：Critical 披露时间：2026年6月2日 漏洞类型：Framework 层整数溢出提权

Google 于 2026 年 6 月发布 Android 安全公告，修复了 124 个漏洞，其中包括一个已被在野利用的零日漏洞 CVE-2025-48595。

漏洞要点：

  • 位置：Android Framework 层
  • 类型：整数溢出漏洞
  • 利用状态：在定向攻击中被利用（商业间谍软件/APT组织）
  • 影响：所有 Android 设备，不依赖特定硬件厂商

值得关注的原因：

  • 商业间谍软件利用：Google 确认该漏洞已被商业间谍软件和 APT 组织在定向攻击中利用
  • 124 个漏洞修复：6 月安全更新是近年来规模最大的 Android 补丁之一
  • Framework 层漏洞：影响所有 Android 设备

防御建议：尽快应用 Android 6 月安全更新，仅从官方商店安装应用，启用 Google Play Protect。

5 严重

CVE-2026-8206 — Kirki WordPress 插件未授权权限提升

CVSS 9.8 受影响版本：Kirki 6.0.0 - 6.0.6 账户接管

Kirki 是 WordPress 上流行的免费页面构建器和自定义器插件。CVE-2026-8206 允许未经身份验证的攻击者通过未检查的密码重置机制，接管包括管理员在内的任何用户账户。

漏洞机制：

  • 类型：未授权权限提升（账户接管）
  • 攻击方式：未检查用户身份直接重置密码
  • 影响：超过百万活跃安装量

值得关注的原因：

  • CVSS 9.8 满分级：未授权 + 网络可达 + 完全接管 = 最高风险组合
  • 影响范围广：Kirki 插件活跃安装量超过百万
  • 攻击门槛极低：无需认证，仅需发送一个密码重置请求即可接管任意账户
  • 已被在野利用：BleepingComputer 报道黑客正在利用此漏洞

防御建议：立即更新 Kirki 插件至 6.0.7 或更高版本，检查 WordPress 用户账户是否异常。

6 高危

Dashlane 密码管理器暴力破解攻击

严重程度：High 披露时间：2026年6月1日 影响：少于 20 个用户的加密保险库被下载

2026年5月31日，密码管理器 Dashlane 遭受暴力破解攻击。攻击者成功绕过了部分账户的双因素认证（2FA），导致少于 20 个个人订阅用户的加密保险库被下载。

攻击过程：

  • 攻击者暴力破解登录尝试
  • Dashlane 检测异常登录并锁定账户
  • 攻击者绕过 2FA
  • 加密保险库被下载

值得关注的原因：

  • 密码管理器安全事件：密码管理器是用户最敏感的数据存储之一
  • 2FA 绕过：攻击者成功绕过双因素认证，说明 2FA 并非绝对安全
  • 加密保险库泄露：虽然保险库是加密的，但攻击者可离线暴力破解弱密码
  • 响应及时：Dashlane 快速检测并锁定账户，限制了损失范围

防御建议：使用强主密码，启用生物识别 2FA，定期检查账户活动。

7 严重

CVE-2026-0826 — HP VoIP 电话关键漏洞可导致企业网络被入侵

CVSS 9.2 披露时间：2026年6月2日（Rapid7 披露） 影响：HP Poly Voice 多款 VoIP 电话

CVE-2026-0826 是 HP Poly Voice VoIP 电话中的一个栈缓冲区溢出漏洞，位于 SIP 协议解析机制中。攻击者可通过发送恶意 SIP 数据包，在受影响设备上以 root 权限执行任意代码。

漏洞机制：

  • 类型：栈缓冲区溢出 → 远程代码执行（root 权限）
  • 攻击方式：发送恶意 SIP 数据包
  • 后果：设备完全控制，可横向移动

值得关注的原因：

  • root 权限执行：成功利用可获得设备完全控制权，包括窃听通话、收集音频生成深度伪造
  • 企业网络入口：VoIP 电话通常位于企业内网，可作为横向移动的跳板
  • 语音深度伪造风险：攻击者可收集高管语音数据，用于生成深度伪造语音进行欺诈
  • Rapid7 披露：知名安全研究机构 Rapid7 发现并披露

防御建议：检查 HP Poly Voice 固件版本，应用安全更新，将 VoIP 电话隔离到独立 VLAN。
