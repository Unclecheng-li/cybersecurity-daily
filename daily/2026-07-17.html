SonicWall SMA1000 KEV 整改大限今日到期：两枚在野漏洞

CVE-2026-15409 SSRF · CVE-2026-15410 代码注入 在野 · 联邦期限 7/17

CISA 于 7/14 将 SonicWall SMA1000 设备的两枚漏洞纳入 KEV，联邦机构整改期限即今日（7/17）。CVE-2026-15409 为未认证服务端请求伪造（SSRF，CWE-918），CVE-2026-15410 为代码注入（CWE-94，管理员→OS 命令），两者均已被确认在野利用，SonicWall 公告为 SNWLID-2026-0008。

漏洞机制/攻击方式：

  • 类型：SSRF（CVE-2026-15409）+ 命令注入（CVE-2026-15410），链式可达设备完全接管。
  • 攻击方式：借 CVE-2026-15409 探测内网/绕过边界 → 以管理员上下文注入 OS 命令 → 获取设备根权限。
  • 影响范围：暴露在互联网的 SonicWall SMA1000 appliances（VPN/远程接入网关）。
  • 后果：网关沦陷 → 内网横向、凭据窃取、持久化，常作勒索前置跳板。

值得关注的原因：

  • 今日即联邦整改大限，未修补设备应直接视为已沦陷，须复盘日志、轮换凭据、排查异常账户与后门。
  • 两漏洞双双在野，与 SonicWall 设备成重点突破点趋势一致（UNC6148/Overstep 先例）。
  • 边界 VPN 网关失守等于交出内网信任边界，缓解优先级高于多数"理论高危"。
攻击者 探测 SMA1000 SSRF 越界 代码注入 设备接管/内网横向

3 高危

LegacyHive：Windows profsvc 零日 PoC 公开（无 CVE 无补丁）

Nightmare-Eclipse · 权限提升 零日 · 未修复

Patch Tuesday 后约半小时，研究员 MSNightmare（Nightmare-Eclipse 身份）公开 LegacyHive PoC：标准用户可滥用 User Profile Service（profsvc，SYSTEM 完整性）的 hive 加载机制，以提升权限挂载任意注册表 hive，实现持久化、凭据窃取与安全产品篡改。该身份此前漏洞已有在野利用记录；截至今日既无 CVE 也无补丁。

攻击方式：诱使 SYSTEM 级 profsvc 加载攻击者控制的 hive 文件 → 注册表级持久化/凭据窃取/防御规避；公开 PoC 被精简且需第二账户，但作者称原始版无此限制。

7 月 Patch Tuesday 创纪录修复约 570–622 CVE。最高评分 VMSwitch CVE-2026-57092（9.9 UAF）允许客户机内代码执行后逃逸至 Hyper-V 宿主；同期 Windows DHCP CVE-2026-50518/56159（9.8 未认证堆溢出，常运行于域控）、RDP CVE-2026-56190、Dynamics CVE-2026-55944（9.8 反序列化）、SharePoint CVE-2026-50522/58644（9.8 RCE）构成关键 RCE 集群。SharePoint 2016/2019 于 7/14 终止扩展支持。
