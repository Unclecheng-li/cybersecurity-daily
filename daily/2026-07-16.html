1 严重

Cursor / AI 编程助手零日：恶意克隆仓库静默触发 Windows 代码执行

CWE-426 · 不受信任搜索路径 零日 · 7 个月未修

Mindgard 于 7/14 完全披露 SpaceX 支持的 AI 编程助手 Cursor（Windows 版）零日：开发者打开项目目录时，编辑器在解析 Git 可执行文件时错误地把"当前工作目录"（不可信工作区）纳入搜索路径。攻击者在克隆仓库根目录放置伪装成 git.exe 的后门，开发者一旦打开项目即被静默执行，无需确认。自 2025-12-15 首次报告起历经 7 个月、70+ 次更新仍未修复，属"不受信任搜索路径"（CWE-426）类通病，同源手法曾致 Git Credential Manager Core 崩溃（CVE-2020-26233）。

漏洞机制/攻击方式：

  • 类型：不受信任搜索路径 / 二进制植入（CWE-426）。
  • 攻击方式：在克隆仓库根目录植入恶意 git.exe → 开发者打开项目目录 → Cursor 在 %PATH% 之前检查当前目录并静默执行。
  • 影响范围：Cursor for Windows（同类问题亦见于 GitHub Copilot CLI、Gemini CLI、Codex 桌面应用，见 Cymulate 6 月研究）。
  • 后果：后门植入、凭据窃取、开发者机沦陷，并借开源仓库投毒形成供应链横向扩散。

值得关注的原因：

  • AI 编程助手成为新的"可执行内容"入口，克隆即中招，开源生态投毒风险陡增。
  • 多家厂商将此类判定为"非漏洞"（Cursor 关闭为 Informative、OpenAI 判 Not Applicable），防御方须将克隆仓库当作可执行内容对待。
  • 与 JADEPUFFER 自主勒索、GhostApproval 等 AI 工具攻击链同源；AWS Kiro 相似漏洞（CVE-2026-10591）已修补，但二进制植入类报告无一修复。
攻击者 植入恶意 git.exe 开发者打开仓库 静默执行 后门/凭据窃取
2 高危

Chromium 内核高危漏洞 CVE-2026-15773（CVSS 9.6，UAF 沙箱逃逸）

CVSS 9.6 · CWE-416 Use-After-Free 全系浏览器 · 已修复

Google 于 7/14 发布 Chrome 150.0.7871.124/.125 修复一处 Chromium 内核 Use-After-Free（CWE-416）漏洞，西澳政府安全运营中心（WA SOC）于 7/16 发布 Critical 级别 advisories（CVE-2026-15773，CVSS 9.6）。该漏洞影响所有 Chromium 内核浏览器：Google Chrome、Microsoft Edge、Brave、Vivaldi。由微软研究员 xinchaotian 于 6/25 报告，7/14 发布修复，7/15 CISA-ADP 给出 9.6 评估。

受影响组件：

组件风险
Google Chrome（150.0.7871.124 之前）严重 · 沙箱逃逸
Microsoft Edge / Brave / Vivaldi严重 · 同源内核
渲染进程 → 浏览器进程中高危 · RCE
3 高危

Keycloak 身份绕过 CVE-2026-1609（JWT 授权授予，已禁用用户可获令牌）

CVSS 8.1 · CWE-284 身份绕过

Keycloak 在启用 jwt-authorization-grant 预览特性且某用户被禁用时，未在该授权授予处理中校验禁用状态。远程低权限攻击者凭外部 IdP 的有效断言令牌，即可为已禁用用户获取 JWT，进而未授权访问受保护资源。CVSS 8.1（红帽 Important / Snyk 8.6），仅影响上游 Keycloak 26.5.2（预览特性默认关闭），RHBK 不受影响；今日（7/16）重新进入 CVE 情报流，值得身份团队立即复查配置。

攻击方式：启用预览特性 → 提交已禁用用户的外部 IdP 断言令牌 → Keycloak 未校验禁用状态即签发该用户 JWT → 持令牌访问受保护端点，绕过身份治理与吊销策略。

4 中高危

Linux 内核 CAN ISO-TP 竞态本地提权（ZDI-26-442，CVSS 7.8）

ZDI-CAN-31764 · CWE 锁缺失 本地提权 · 已修复

ZDI 于 7/15 公开 Linux 内核 CAN ISO-TP 协议 socket close 处理中的竞态条件本地提权漏洞（CVE 待分配，ZDI-CAN-31764，CVSS 7.8）：本地低权限攻击者利用对象操作期间缺少 Proper Locking 的竞态，可提权至内核态执行任意代码。Linux 已发布修复（6/7 起内核补丁）。CAN 总线广泛用于汽车与工业 OT，内核 CAN 子系统漏洞直接影响车载与工控安全。

5 中高危

日本食品冷链巨头 Nichirei 确认遭网络攻击（7/16 披露）

东证 Prime 2871 · 冷链停摆 安全事件

日本食品巨头 Nichirei（日冷）于 7/16 公告：7/13 系统故障经调查确认为网络攻击所致，服务器遭入侵，部分含个人信息，已向个人信息保护委员会提交初步报告。为防损害扩大，7/13 起断开集团系统，导致冷藏仓库（Nichirei Logistics）进出口、Nichirei Foods 冷冻食品出货受阻；计划 7/17 起在外部专家协助下逐步恢复。与近期 KDDI、Nihon Kotsu 等日本企业连环事件共振。
