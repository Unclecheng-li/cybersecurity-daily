XRING — 阿里 XQUIC QUIC/HTTP/3 库一行错误潜伏 4 年零补丁：~260B 合规 QPACK 流量远程 DoS 主流 HTTP/3 服务，4 次跟进阿里无回应

阿里 Tengine（淘宝 / 支付宝 / 阿里云 CDN）依赖默认 QPACK 设置均暴露，缓解只能"禁用 HTTP/3 或关闭 QPACK 动态表"——2026 年"被忽视的大厂开源组件"典型治理失效。

1 严重

XRING — 阿里 XQUIC 库一行错误代码潜伏 4 年零补丁，260B 合法流量远程 DoS HTTP/3 服务

FoxIO 研究员 Sébastien Féry 7/8 公开 / 阿里至 7/10 无回应 无 CVE 无补丁 远程 DoS

FoxIO 研究员 Sébastien Féry 7/8 公开 XRING——阿里开源 QUIC / HTTP/3 库 XQUIC 的 QPACK 表扩容代码中"一行变量用错"，导致 size_t 整数下溢 + 巨型 memcpy 跨内存写入。攻击者只需发送 ~260 字节完全合规的 QPACK 流量（64→65 字节 grow），命中特定的环形缓冲区换行布局即可让服务端进程崩溃——无需登录、无需畸形包、无需畸形值。

关键事实：

  • 影响范围：所有 XQUIC 版本（自 2022/1 首次发布到 v1.9.4，含最新版）
  • 下游：阿里基于 Nginx 的 Tengine（淘宝、支付宝、阿里云 CDN 在用）
  • 披露时间线：4/7 通过 XQUIC 安全政策向阿里报告 → 5/9 前 5 次跟进无回应 → 7/8 公开 → 7/10 仍无补丁、无 CVE
  • 临时缓解：将 SETTINGS_QPACK_MAX_TABLE_CAPACITY 设为 0（关闭 QPACK 动态表）或完全禁用 HTTP/3
  • 相关历史：三周前 NGINX HTTP/3 模块曝 CVE-2026-42530（UAF 同 QPACK encoder stream 攻击面）；6 月 HTTP/2 Bomb 系列（HPACK 滥用）打穿 Nginx/Apache/IIS/Envoy

值得关注的原因：

  • 潜伏 4 年（2022/1 起）+ 4 次沟通无回应 + 公开日仍无补丁——这是 2026 年"开源 + 大厂"组件治理失效的标杆案例
  • 阿里云生态（含电商、CDN、API 网关）凡开启默认 QPACK 的 HTTP/3 端点均暴露——与 CUPS 系列 2024-2026 反复出现的"被忽视组件"同根病灶
  • XRING 与 6 月 HTTP/2 Bomb、NGINX CVE-2026-42530 形成 HTTP/2/HTTP/3 header 压缩逻辑已成 RCE/DoS 高发面——任何 HTTP/3 部署方都应将"压缩逻辑补丁"纳入紧急变更窗口
  • 缓解优先级：① 立即禁用 HTTP/3；② 如不能禁用，至少关闭 QPACK 动态表；③ 在外暴露的阿里云 CDN / API 网关前加 WAF / 反向代理层；④ 监控 XQUIC 仓库后续 30 天内的 release 与 CVE 公告
260B QPACK 合规流量 XQUIC QPACK 扩容 size_t 下溢 远程 DoS

MODBEACON — 银狐/Ghost 分发商以 Rust + gRPC 升级工具栈，科技 / 教育 / 国企成定向靶标

奇安信红雨滴 7/7 公开 / 中国相关团伙 Silver Fox / UTG-Q-1000 定向 APT 投放

奇安信威胁情报中心红雨滴团队 7/7 公开 MODBEACON——一个 Rust 编写的模块化 RAT，归属于中国相关网络犯罪团伙 Silver Fox / UTG-Q-1000（Ghost 分发商）。该分发商使用 Gh0st RAT + WinOS / ValleyRAT 变种 + 新 MODBEACON 进行"分发 + 投放"分工，是典型的 "军火商 + 流量中间商"复合体。

核心能力：

  • 语言：Rust（规避 AV/EDR 静态签名）
  • 部署形态：内存态 stage/implant，无文件落盘
  • 通信：gRPC over HTTP/2 + TLS，复用 Xray/V2Ray 翻墙框架的传输层作 C2 通道
  • 持久化：服务 + 计划任务 + WMI 永久订阅事件（CBPUserTimer 定时器）
  • 插件架构：native-v3 插件（entry/init/fini RVA），按需下发
  • C2 基础设施：Amazon + Cloudflare CDN
  • 目标：中国境内科技、教育、国企；外延柬埔寨博彩"黑吃黑"

6. U-Boot 引导加载程序六连漏洞 — 2 个可在 FIT 数字签名校验前执行代码

Binarly 公开 U-Boot 6 个新漏洞（4 个 DoS + 2 个代码执行），攻击者把恶意 FIT（Flattened Image Tree）镜像送入引导阶段，在 U-Boot 校验数字签名之前触发——影响家用路由器、智能摄像头、服务器 BMC 管理芯片。所有基于 U-Boot 的边缘设备本周内必须升级固件。

7. Injective Labs SDK 供应链投毒 — 17 个 @injectivelabs/sdk-ts@1.20.21 包窃钱包私钥

Injective Labs 官方 GitHub 仓库 7/8 被劫持，攻击者通过"有合法贡献历史"的开发者账号提交恶意 commit，污染 @injectivelabs/sdk-ts@1.20.21 + 17 个 @injectiveLabs/* 包的同版本，伪装"遥测"功能窃加密货币钱包私钥与助记词；GitHub 上对应 release 制品仍可下载——所有 Injective 生态 DApp / 钱包 / 索引器应立即锁版本。

8. Tangem 硬件钱包激光攻击重置密码 — 无补丁，已售出卡片均受影响

Ledger Donjon 证实可通过精确时序激光脉冲注入 Samsung S3D232A 芯片重置密码为攻击者自定义值（无需旧密码、无需备份卡），成本约 25 万美元实验室设备，需物理接触 + 切开卡片外壳。Tangem 卡片无软件更新能力，所有已售出卡片均受影响——卡片已丢失或被盗且金额较大的用户应立即转移资产。

9. "Ill Bloom" 加密钱包弱熵漏洞 — 5 月以来已造成 500 万美元+损失

Coinspect 公开部分老旧 / 小众钱包（浏览器扩展和移动 App，部分可追溯至 2018 年）助记词生成时随机数熵不足，攻击者可暴力枚举并还原私钥。已确认损失超 500 万美元（5/27 协调攻击窃取约 310 万美元，涉及 431 个钱包；后续 210 万美元 USDT 被盗）。硬件钱包与主流软件钱包不受影响。

10. BlackCat 勒索软件谈判员获刑 70 个月 — RaaS 生态内部信任崩塌样本

佛罗里达州 Angelo Martino（41岁）被联邦法院判处 70 个月（近 6 年）监禁——他作为勒索软件谈判顾问代表 5 名受害方的同时，向 BlackCat/ALPHV 泄露受害方谈判策略与底价，被法官定性为"双面间谍，最大化客户伤害与网络犯罪分子的经济利益"。典型 RaaS 生态内部信任崩塌案例——所有聘请第三方谈判方的受害企业需重新评估信任模型。

11. 281 款免费 Android VPN 应用泄露用户数据 — 累计安装超 24 亿次

密歇根大学 / 新墨西哥大学 / IIT Delhi 联合发布 MVPNalyzer 报告（2 月 NDSS 安全会议公开）：281 款 Google Play 热门免费 VPN 应用中，29 款泄露 DNS 等用户流量 / 61 款明文传输部分数据 / 5 款明文传输配置文件——任何使用免费 VPN 的 Android 用户应立即卸载并改用企业级方案。
